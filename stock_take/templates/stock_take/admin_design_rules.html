Design Principles
This is the "Atlas Cobalt" design system — a Sliderobes-branded rebrand of the Atlas platform's colours, type, and shape language. See obsidian/CSS Design System.md for the full written reference.
1. Spacing Rhythm — All padding, margin, and gap values must use the scale below (4, 8, 12, 14, 18, 22…). Never invent an arbitrary pixel value.
2. Shape Language — Buttons and inputs use var(--radius-sm) (9px rounded rectangles — the square button standard); cards and containers use var(--radius-md) (14px). Badges and status pills use var(--radius-pill), which is now squared off (points at --radius-sm) so pills read as rectangular buttons rather than fully-rounded chips. var(--radius-round) (999px) is reserved for genuinely round elements only — toggle-switch tracks and circular dots/indicators.
3. Token-only Colours — Never hardcode hex colours in page-specific CSS. Always use var(--name) tokens. This ensures dark/light mode works everywhere.
4. Surface Hierarchy — Page background (--bg-primary) → Cards (--bg-secondary) → Inset areas (--bg-tertiary). Sidebar and topbar sit on --bg-nav.
5. Semantic Colours — Use the right colour for the right meaning: primary/accent (action), success (positive), warning (caution), danger (destructive), info (neutral info).
6. Two-font System — Chakra Petch (display) for headings and numeric stat values; Manrope (body) for everything else.
7. No inline styles — Page styles live in a matching static/css/<page>.css file, never in a <style> block or a style="" attribute.

Form Controls
Standard inputs, selects, and textareas.
Usage: Use .form-control or the element selectors. Inputs use --bg-input background and --radius-sm corners. Focus state shows --primary-color border + subtle glow.
Checkboxes: One style site-wide, defined on the bare input[type="checkbox"] selector in styles.css — 16px, --radius-xs corners. Never restyle a checkbox in a page CSS file; to recolour one, set --checkbox-color on the instance and hover/checked/focus all follow. accent-color has no effect (the base style sets appearance: none).
Number inputs: The shared .qty-stepper-input −/+ pill is for quantity fields only. base.html auto-wraps a number input when it reads as a quantity (a .quantity-input/.qty-step-value class, the board cnt column, or a qty/quantity/count token in its name/id/class). Every other number input (dimensions, prices, rates, VAT, discount…) stays a plain input. Native browser spinners are hidden globally. Add no-pnx-spinner to force a quantity-named field to stay plain.

Quick Reference
When adding a new page:
1. Extend base.html
2. Use .card for content sections
3. Use .btn .btn-* for actions
4. Use .badge .badge-* for status indicators
5. Use .table for data
6. Use .alert .alert-* for messages
7. Use .form-control for inputs
8. Only use CSS variables — never hardcode hex colors
9. All spacing must use the --sp-* scale
10. Put page styles in static/css/<page>.css — no <style> blocks or style="" attributes
CSS files:
• styles.css — Global tokens + all shared components
• light-mode.css — Variable overrides for body.light-mode
• Page-specific CSS — Only for unique page layouts, never for redefining base components
