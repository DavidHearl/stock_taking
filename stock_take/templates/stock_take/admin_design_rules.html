Design Principles
1. 4px Grid — All padding, margin, gap, and spacing values must be multiples of 4 (4, 8, 12, 16, 20, 24…). Never use 5px, 7px, 10px, 15px etc.
2. Pill Buttons — All interactive buttons use border-radius: var(--radius-pill). Containers, cards, and inputs use var(--radius-sm) or var(--radius-md).
3. Token-only Colours — Never hardcode hex colours in page-specific CSS. Always use var(--name) tokens. This ensures dark/light mode works everywhere.
4. Surface Hierarchy — Page background (--bg-primary) → Cards (--bg-secondary) → Inset areas (--bg-tertiary). Sidebar and topbar sit on --bg-primary.
5. Semantic Colours — Use the right colour for the right meaning: primary (action), success (positive), warning (caution), danger (destructive), info (neutral info).

Buttons
All buttons use pill shape. Use filled buttons for primary actions, outline for secondary, ghost for icons/tertiary.
Filled Buttons
Outline Buttons
Ghost & Small Buttons
Attention Pulse (draw the eye to important actions)
Hover: All filled and outline buttons get a coloured box-shadow glow on hover. No vertical movement.
Pulse: Add .btn-pulse-primary (or success/danger/warning) to make a button pulse — use sparingly to draw attention. Pulse stops on hover.
Usage: .btn .btn-primary for main CTA. .btn .btn-outline-* for secondary. .btn .btn-ghost for minimal/icon actions. Add .btn-sm for compact areas like table rows.

Quick Reference
When adding a new page:
1. Extend base.html
2. Use .card for content sections
3. Use .btn .btn-* for actions
4. Use .badge .badge-* for status indicators
5. Use .table for data
6. Use .alert .alert-* for messages
7. Use .form-control for inputs
8. Only use CSS variables — never hardcode hex colors
9. All spacing in multiples of 4px
CSS files:
• styles.css — Global tokens + all shared components
• light-mode.css — Variable overrides for body.light-mode
• admin_pages.css — Admin-specific styles
• Page-specific CSS — Only for unique page layouts, never for redefining base components
