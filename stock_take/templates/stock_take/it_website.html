Sliderobes Website

Set up the WordPress site locally, then follow the routine for updating a plugin and pushing your changes back to everyone.

First-time setup

Do this once to get the site running on your machine. Already set up? Skip straight to Updating a plugin below.

Before you start. These commands are for Windows PowerShell. Run each block in order. If a winget command isn't recognised, download the app from the linked site instead — the result is the same.
  1. 1

    Install Visual Studio Code

    The code editor you'll use to browse and edit the site. Install it with the command below, or download it from code.visualstudio.com. 

    winget install --id Microsoft.VisualStudioCode -e
  2. 2

    Install Git

    Git is the tool that downloads (clones) the website's code and lets you pull future updates. Install it, then close and reopen PowerShell so the git command becomes available. 

    winget install --id Git.Git -e

    Set your name and email once so your changes are attributed correctly: 

    git config --global user.name "Your Name"
git config --global user.email "you@sliderobes.com"
  3. 3

    Create a GitHub account

    GitHub is where the website's code lives online. You need a free account to download the code and to push your changes back up. Already have one? Skip to the next step.

    1. Go to github.com/signup.
    2. Enter your Sliderobes email, then choose a password and a username (for example yourname-sliderobes).
    3. Complete the puzzle to prove you're human, then click Create account.
    4. Open the verification email GitHub sends you and click the link (or type the code) to confirm your address.
    5. If it asks which plan you want, pick the Free plan — it's all you need.
    6. Send your new username to David so he can grant you access to the sliderobes repository.
  4. 4

    Clone the repository

    This downloads the site into a sliderobes folder inside your Documents. You'll be prompted to sign in to GitHub the first time — use your GitHub account (or a personal access token if the repo is private). 

    cd $HOME\Documents
git clone https://github.com/DavidHearl/sliderobes.git

    To pull the latest changes later, run git pull from inside the sliderobes folder.

  5. 5

    Set up a local WordPress environment

    The site is WordPress/PHP, so it needs PHP, MySQL and a web server to run. The easiest option is Local (by WP Engine) — it bundles everything and creates a local site in a few clicks. Install it with: 

    winget install --id WPEngine.Local -e

    In Local, create a new site, then point its app/public (or wp-content) folder at the files you cloned in step 3 — or copy the cloned theme/plugin folders into the new site's wp-content. Prefer XAMPP instead? Install PHP + MySQL with XAMPP and serve the folder from htdocs.

  6. 6

    Open the project in VS Code

    Open the cloned folder in VS Code to start editing. When prompted, install the recommended PHP extension for syntax highlighting and hints. 

    code $HOME\Documents\sliderobes

    Recommended extensions — install from the Extensions panel (Ctrl+Shift+X) or run: 

    code --install-extension bmewburn.vscode-intelephense-client
code --install-extension wordpresstoolbox.wordpress-toolbox

Updating a plugin

This is the everyday routine for getting a plugin change onto the live site. Do the steps in order, every time — most problems come from skipping one.

Golden rule: pull before you touch anything. Always run git pull first to grab everyone else's latest work. Skipping this is the number-one cause of conflicts.
  1. 1

    Open the project

    Open the sliderobes folder in VS Code (File → Open Folder), or run the command below. Every command after this goes in VS Code's built-in terminal — open it with Terminal → New Terminal. 

    code $HOME\Documents\sliderobes
  2. 2

    Pull the latest changes

    This updates your copy with anything other people have pushed since you last worked. Do this every single time before you make a change. 

    git pull
  3. 3

    Drop in the updated plugin

    In the file list on the left of VS Code, open wp-content → plugins. Swap the old plugin for the new version:

    1. Right-click the plugin's existing folder and choose Delete.
    2. Drag the new plugin folder from File Explorer straight into the plugins folder in VS Code.
    3. Keep the folder name exactly the same as before — don't add "(1)" or a version number to it.

    Only given a few loose files instead of a whole folder? Drag those in and choose Replace when it asks.

  4. 4

    Check what you changed

    Click the Source Control icon on the left bar (or press Ctrl+Shift+G). You'll see a list of every file you touched — click any one to see the before/after. If something looks wrong, hover the file and click the Discard (↩) arrow to undo it before it goes anywhere.

  5. 5

    Save a commit

    A commit is a saved snapshot with a short note describing what you did. Stage every change and commit it: 

    git add .
git commit -m "Update Contact Form plugin to v2.4"

    Change the text in quotes to describe your actual change — keep it short and clear so the next person knows what happened.

  6. 6

    Push to GitHub

    This uploads your commit so it's live for everyone. You may be asked to sign in to GitHub the first time — use the account from the setup steps above. 

    git push

    That's it — your change is now on GitHub. Refresh the site to see it.

If git push is rejected saying the remote has changes, run git pull again and then git push. If you ever see the word CONFLICT, stop — don't guess. Raise a ticket and someone will help you sort it out.
Stuck on any step? Raise a ticket in Atlas and include the exact command you ran and the error message.
